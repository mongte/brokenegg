# 페이지 레이아웃 패턴 — 신규 페이지 작성 기준

새 페이지(상세/하위 페이지)를 **랜딩과 같은 톤**으로 만들 때 따르는 골격과 규칙. 전용 스킬 `/brokenegg-page`가 이 노트를 기준으로 동작한다.

> 상위: [[디자인 시스템 - 개요]] · 토큰: [[디자인 시스템 - 디자인 토큰]] · 컴포넌트: [[디자인 시스템 - 컴포넌트]]

## 페이지 골격 (필수)
모든 페이지는 동일한 셸을 공유한다:

```
<grid-bg />              ← 고정 배경 격자 (z-index:-1)
<div class="canvas">     ← 흰색 라운드 카드 (max 1440, radius 32)
  <SiteHeader />         ← 재사용
  ── 페이지 본문 섹션들 ──
  <FinalCta />           ← (선택이지만 권장) 전환 밴드
  <SiteFooter />         ← 재사용
</div>
```

- 본문은 위에서 아래로 섹션을 **쌓고**, 각 섹션 경계는 `border-top: 1px solid var(--border)`.
- 섹션 좌우 패딩 기본 **40px**. 큰 호흡이 필요한 섹션은 세로 80~120px.

## 섹션 빌딩블록 (레고처럼 조합)
페이지 성격에 맞게 아래에서 골라 조합한다. (클래스/위젯은 [[디자인 시스템 - 컴포넌트]])

1. **Hero (페이지 도입)** — 세리프 `section-tag` 한 단어(예: `Solution`, `About`, `Detail`) + 큰 sans 헤드라인 + 필요 시 CTA. **모든 페이지는 hero로 시작**해 맥락을 준다.
2. **태그/정렬 바** — `.section-header`(좌측 태그들 + 우측 보조 텍스트). 목록/카테고리 페이지 도입부.
3. **텍스트 그리드** — `.problem-grid`형 N열 카드. 특징·단계·스펙 나열에.
4. **미디어 갤러리** — `.services-grid`형 풀블리드 검정 배경. 페이지당 1회 권장. 이미지 그레이스케일→호버 컬러.
5. **인용/후기** — `.testimonials` 세리프 2열.
6. **본문 산문 블록** — 좁은 폭(`max-width: 720~900px`) 텍스트. 상세 설명용(신규 패턴이면 토큰 준수해 추가).
7. **Final CTA** — `.final-cta` 중앙 정렬 큰 제목 + 버튼. 푸터 직전.

## 리듬 규칙
- **밝음 → 밝음 → (검정 갤러리) → 밝음** 식으로 페이지당 한 번의 어두운 면으로 리듬을 준다.
- 연속 텍스트 섹션이 많으면 중간에 갤러리/인용으로 호흡.
- 한 페이지에 hero는 하나. section-tag(72px 세리프)는 페이지 최상단에만.

## 카피·콘텐츠
- 문구는 추측하지 말고 [[랜딩 페이지 카피]]·[[메인 카피]] 및 해당 솔루션 노트([[솔루션 - 3D 디지털제품여권 (DPP)]] 등)에서 가져온다.
- 위키에 없으면 `status: review`로 표시하고 사용자 확인. 톤 규칙(클리셰 금지)은 [[랜딩 페이지 카피]] 상단 참조.

## FSD 배치 (신규 페이지)
| 만들 것 | 위치 |
|---|---|
| Next 라우트 | `src/app/<route>/page.tsx` (얇게 — view만 렌더) |
| 화면 조립 | `src/views/<name>/ui/<Name>Page.tsx` + `index.ts` |
| 이 페이지 전용 블록 | 위 view 내부 (재사용되면 `src/widgets/`로 승격) |
| 새 도메인 명사 | `src/entities/<name>` |
| 공용 프리미티브 | `src/shared/ui/<name>` |
| 스타일 | 기존 클래스 재사용 우선 → 부족하면 `global.css`에 **토큰 기반** 클래스 추가 |

import 방향: `app → views → widgets → features → entities → shared` (아래로만). 공개 API는 슬라이스 `index.ts`. → [[fsd-feature-sliced-design]] · [[fsd-public-api]]

## 메타데이터
- 페이지마다 `metadata`(title/description)를 라우트 또는 view에서 설정. 사이트 공통 값은 `src/shared/config/site.ts` 참고.
