# 컴포넌트 카탈로그

전역 클래스(`global.css`)와 FSD 컴포넌트의 대응표. **신규 페이지는 새 클래스를 만들기 전에 여기 있는 것을 먼저 재사용한다.**

> 상위: [[디자인 시스템 - 개요]] · 토큰: [[디자인 시스템 - 디자인 토큰]] · 배치: [[디자인 시스템 - 페이지 레이아웃 패턴]]

## 재사용 프리미티브 (`src/shared/ui`)
| 컴포넌트 | 클래스 | 변형/Props | 설명 |
|---|---|---|---|
| **Badge** | `.badge`, `.badge.active` | `active`(검정 반전), `className`, `style` | 알약형 마이크로 라벨. 기본 회색 배경/뮤트 텍스트. |
| **Button** | `.btn`, `.btn-primary`, `.btn-secondary` | `href`, `variant`(primary/secondary) | 링크형 CTA(`<a>`). primary=검정 채움, secondary=테두리. |
| **Logo** | — | `variant`(dark/light), `height` | 흰색 PNG를 라이트 배경에선 `brightness(0)`로 검정 렌더. |

| **SiteFooter** | `footer` | 카피라이트·링크·레퍼런스 코드 | ✅ 모든 페이지 그대로 |

## 상호작용 패턴 — 인라인 프리뷰 (`src/features/service-preview`)
서비스 카드 클릭 → 아래로 슬라이드다운하며 iframe(라이브 3D / 유튜브) 노출.
- 핵심 클래스: `.service-preview(.open)`, `.service-preview-inner/-bar/-label/-close/-stage`
- 규칙: **슬라이드 완료 후에만** 노출(`.is-revealed`), 닫히면 `display:none`로 park(인스턴스 유지·재로딩 없음).
- 콘텐츠 종류 라벨: `previewKind: 'live3d' | 'video'` → 카드 큐 `LIVE 3D ▸` / `WATCH ▸`, 패널 라벨 `· LIVE 3D` / `· VIDEO`.
- 신규 페이지에서 "클릭하면 펼쳐지는 미디어" 패턴이 필요하면 이 피처를 재사용/참고.

## 상호작용 패턴 — 데모 요청 모달 (`src/features/demo-request`)
모든 "데모 요청" CTA(헤더·hero·final-cta)는 `DemoRequestButton`이며, 클릭 시 **공유 모달**을 연다(앵커 스크롤 아님).
- 열기 신호: `openDemoModal()`(`lib/events.ts`, window CustomEvent) → `DemoRequestModal`이 수신. 모달은 `layout.tsx`에 한 번만 마운트.
- 폼 필드: 회사명*, 이름, 연락처*, 이메일, 문의 내용 + **개인정보 수집·이용 동의**(필수, 체크해야 전송 활성화). 제3자 제공 없음. 허니팟(`.demo-honeypot`)으로 봇 차단.
- 동의서 원문은 `model/policy.ts`(`PRIVACY_CONSENT`)에 분리. 이 폼이 실제 수집하는 범위만 간결 고지(수집항목·목적·보유기간 1년·거부권). `.demo-policy`(`max-height:150px` + `overflow-y:auto`, `white-space:pre-line`) 박스로 노출.
- 전송: `POST /api/demo-request` → 서버가 **Web3Forms**로 전달해 지정 메일로 발송. access_key는 `WEB3FORMS_KEY`(서버 env, 번들 비노출).
- 검증/예외처리: `lib/validate.ts`의 `validateDemoForm`(필수·길이·이메일/연락처 형식)을 **클라이언트(즉시 피드백)·서버(방어) 양쪽** 공유. 필드별 에러는 `.demo-field-error`(흑백, 에러 시 2px 밑줄). 전송엔 15초 타임아웃(`AbortController`), 네트워크/타임아웃/서버오류 메시지 분기, 전송 중 중복 제출·닫기 차단.
- 스타일: `.demo-modal-backdrop/.demo-modal`(radius-lg, blur 백드롭), 입력은 **밑줄형**(border-bottom, focus 시 검정). 컬러 액센트 없음, 토큰만 사용.
- 신규 페이지에서 문의/신청 폼이 필요하면 이 피처를 재사용한다.

## 새 컴포넌트를 만들 때 규칙
1. **토큰만 사용**(색·반경·서체·간격). 하드코딩 색/픽셀 반경 금지.
2. 세그먼트 이름은 목적 기반(`ui/model/api/lib/config`). `components/hooks/types` 금지. → [[fsd-feature-sliced-design]]
3. 마이크로 라벨은 항상 대문자·700·letter-spacing.
4. 강조는 색이 아니라 **굵기/크기/세리프/여백**으로.
5. 단일 페이지에서만 쓰면 그 view 안에, 여러 페이지에서 쓰면 widget으로 승격.
