# 핵심 기술: 3D 웹뷰어 배포 아키텍처 (Godot × Next.js)

[[핵심 기술 - 3D 인터랙션 시뮬레이션 엔진]]으로 만든 3D 콘텐츠는 **Godot 4 웹(HTML5/WASM) 익스포트**로 출력되어, brokenegg-app(Next.js 16, Vercel) 위에서 **QR별 개별 라우트**로 서빙됩니다. 이 노트는 그 배포 방식의 **단일 출처(SSOT)**입니다. [[솔루션 - 3D 디지털제품여권 (DPP)]]의 QR→3D 뷰어 경험이 이 구조 위에서 동작합니다.

> 2026-06-24 딥 리서치로 도출 → **같은 날 brokenegg-app에 구현·렌더 검증 완료**(아래 "✅ 검증된 구현"). §1~§8 은 리서치 가설(일부 정정됨), 정답은 "검증된 구현"+§9~§11.
> 2026-06-28 **프로덕션(R2) 경로 구현 완료** — 로컬 라우트 핸들러를 Cloudflare Worker(`workers/godot-viewer/`)로 포팅, R2 업로드 스크립트(`godot-assets/upload-r2.mjs`), Next.js env 스위치(`NEXT_PUBLIC_GODOT_CDN_BASE`) 추가. **실제 배포는 Cloudflare 인증/버킷 생성(사용자 자격증명) 대기.** 아래 "프로덕션(R2)" 섹션 참고.

## 0. 핵심 요구사항
- **비용 최우선**: 40MB급 wasm을 QR 스캔으로 반복 다운로드 → 전송비(egress)가 비용을 지배.
- **QR별 독립**: QR 하나 = 3D 렌더 하나, 각 export 따로 관리. 새 제품 추가 시 **재배포 없이** 운영.
- **빌드에 안 담기**: 40MB×N을 Next.js `public/`/번들에 넣으면 배포 한계·빌드 비대화.

## ✅ 검증된 구현 (2026-06-24 · 실측 + Playwright) — **이 섹션이 최신·정답**
brokenegg-app 안에 구현·렌더 검증 완료. `/ko/viewer/poim` 에서 파란 3D 모델 렌더 확인(헤드리스 Chrome 스크린샷 + Playwright e2e). **아래 §1~§8 은 초기 딥리서치 가설이며, 충돌 시 이 섹션 + §9~§11 이 우선.**

### 실측으로 확정된 핵심 사실 (가설 정정 포함)
1. **이 Poim export 는 single-thread 빌드** (`GODOT_THREADS_ENABLED=false`, 엔진 로그 "single-threaded") → **COOP/COEP 불필요**, `next.config.js` headers() **제거함**. (Playwright: `crossOriginIsolated=false` 로 정상 렌더) ↔ §2 의 "현재 export 멀티스레드" 는 틀림.
2. **필수 파일 5개**: `viewer.js` · `viewer.wasm(.br)` · `viewer.pck(.br)` · **`viewer_function.js`** · **`poim_scene.zip`**.
   - `viewer.pck` = **공유 베이스 씬(뷰어 앱)**. **QR별로 다른 건 `poim_scene.zip`(실제 모델 데이터, 33MB)** — 런타임에 Godot `HTTPRequest` 로 다운로드. ↔ §1·§3·§7 의 "QR별 `.pck` 교체" 는 틀림 → **zip 교체가 맞음**.
   - `viewer_function.js`(전역 `prograssbar`/`downloadFileLength`+`initSetting`) 와 zip 을 빠뜨리면 엔진은 떠도 **씬이 안 뜸**.
3. **임베드 = canvas 직접 마운트, auto-start 방식**: viewer.js 가 `#canvas`/`#status` DOM 을 찾아 자동 부팅. **canvas override(§4)는 안 씀.** 동적 주입 `<script>` 는 `document.currentScript=null` 이라 경로 오해석 → **핸들러가 viewer.js 의 `executable` 을 절대경로로 치환**해 해결. `viewer_function.js` 의 `initSetting.url` 은 **전체 URL(scheme+host)**로 치환(HTTPRequest 요구).
4. **brotli 는 됨 — 단 `Content-Length` 없이 chunked 로**: Godot 로더가 `new Uint8Array(Content-Length)` 로 버퍼 선할당 → 사전압축본을 Content-Length 와 함께 주면 디코드크기와 안 맞아 **오버플로("Failed loading file")**. ✅ `ReadableStream`(Content-Length 생략, chunked)으로 보내면 **압축 전송(wasm 37.7→7.5MB, pck 3.4→1.1MB) + 로더 호환**. ↔ §6 "Godot 이 streaming 사용" 은 틀림.
5. **Godot 은 `instantiateStreaming` 안 씀** — 진행바(`response.body` reader)를 위해 **비스트리밍**(다 받고 `WebAssembly.instantiate`). → V8 코드캐시·streaming 이점 없음. **다운로드 % 는 Content-Length 필요** → chunked(압축) 서빙이면 불확정 바(트레이드오프).
6. **teardown(메모리)**: 한 탭 다중 SPA 전환 대비 — 언마운트 시 `window.__godotEngine.requestQuit()` + `WEBGL_lose_context` + 주입 스크립트 제거(핸들러가 엔진 전역 노출). QR=매 스캔 새 페이지면 전체 리로드로 자동 해제라 무관.
7. **원본 `wasm`/`pck` 삭제 가능**: `.br` 만 두면 핸들러가 br 미수락 클라이언트엔 `.br` 즉석 해제로 폴백. zip 은 무압축(이미 압축, br 1.4%).

### 프로덕션(R2) — 구현·검증 완료 (실배포만 자격증명 대기) ✅ 2026-06-28
로컬 같은 오리진 핸들러를 **Cloudflare Worker × R2** 로 포팅 후 **`wrangler dev`(로컬 R2 바인딩)로 end-to-end 렌더 검증 완료** — 헤드리스 Chromium 에서 Worker(:8799)가 서빙한 에셋만으로 Poim 3D 모델 렌더 성공(실패요청 0, WebGL2, `crossOriginIsolated=false`). 클라이언트(QR 폰)가 **Worker 에서 직접** 받아 R2 egress($0) 유지(Next.js 프록시 ✗ — 그러면 Vercel egress 과금). br 은 **Worker 가 Content-Length 생략 chunked 로**(정적 `.br` 직접 서빙은 오버플로 재발). CORS 필수, `initSetting.url`·`executable` 은 Worker 가 전체 URL 로 치환(로컬 핸들러와 동일 로직).

> **⚠️ 실측으로 발견한 핵심 함정 — `encodeBody: 'manual'`**: Cloudflare/workerd 런타임은 응답을 `Accept-Encoding` 기준으로 **자동 재압축**한다. 사전압축 `.br` 본문을 그대로 주면서 `Content-Encoding: br` 만 달면, 런타임이 그 위에 br 을 **한 번 더** 입혀 `br(br(wasm))` 이중 인코딩이 된다 → 브라우저는 1회만 풀어 Godot 에 still-br 을 넘김 → "Failed loading file". 해결: 사전압축 응답은 반드시 `new Response(body, { headers, encodeBody: 'manual' })` 로 — "본문은 이미 인코딩됨, 재압축 말라". `wrangler dev` 로 이중 인코딩 재현→수정 검증(받은 바이트가 원본 `.br` 과 정확히 일치, 1회 디코드 시 37.7MB 유효 wasm). (로컬 `route.ts`는 Node 런타임이라 자동 재압축이 없어 이 함정이 없었음 — Worker 전용 이슈.)

**구현 위치:**
- Worker: `workers/godot-viewer/src/index.js`(route.ts 의 R2 버전) + `wrangler.toml`(버킷 `brokenegg-godot`) + `README.md`(배포 절차).
- 업로드: `apps/brokenegg-app/godot-assets/upload-r2.mjs` — 서빙 5종을 `r2://brokenegg-godot/<id>/` 로(`wrangler r2 object put`).
- Next 스위치: `viewer/[id]/page.tsx` 가 `NEXT_PUBLIC_GODOT_CDN_BASE` 있으면 Worker(cross-origin), 없으면 로컬 `/godot` 핸들러로 폴백.

**배포(사용자 자격증명 필요):** `cd workers/godot-viewer && npm i && npx wrangler login && npx wrangler r2 bucket create brokenegg-godot && npx wrangler deploy` → 출력 URL 을 `NEXT_PUBLIC_GODOT_CDN_BASE` 에 → `node godot-assets/upload-r2.mjs poim`. 상세: `workers/godot-viewer/README.md`.

**키 분리 운영:** 엔진(viewer.wasm/.pck.br)·뷰어 JS 는 QR 공유, QR별로 다른 건 `<id>/<scene>.zip`(모델). 새 모델 = R2 에 폴더 1개 업로드 + `ASSET_FOLDERS` 한 줄 → **재배포 불필요**. egress 무료라 33MB zip 을 많이 서빙해도 비용 0 — **이게 R2 선택의 핵심**.

## 1. 결정 요약 (decision) — *초기 리서치 요약 (정정은 위 "검증된 구현" 우선)*
| 축 | 결정 | 근거 |
|---|---|---|
| 멀티스레드 | **끈다 (single-thread)** — *이 export 는 이미 single-thread* | Godot 4.3+ 기본·권장. COOP/COEP 문제 소멸 |
| 에셋 호스팅 | **Cloudflare R2** (외부 object storage) | egress $0 — 반복 전송 비용 지배에 결정적 |
| 임베드 | **자기 라우트에 canvas 직접 마운트** (iframe ❌) | cross-origin iframe isolation 회피, same-origin 단순 |
| QR별 데이터 | **wasm·pck 공유 + QR별 `poim_scene.zip`(모델) 교체** *(§1 원문 "pck 교체"는 정정됨)* | 엔진 1회 캐시 + QR별 데이터만 다름 |
| 로딩 | Brotli(chunked) + immutable 캐시 + 라우트 진입 시 lazy load | 콜드 로딩 비용·체감속도 |

## 2. 멀티스레드 끄기 — 가장 큰 단순화 레버
현재 export는 `SharedArrayBuffer`(멀티스레드) 사용 → 이것이 COOP/COEP cross-origin isolation을 *전부* 요구하는 근본 원인.
- **Godot 4.3+ single-thread 웹 익스포트가 "기본이자 권장" 모드** — 공식 표현 *"the pesky COOP/COEP headers 없이"* 배포 가능. (godotengine.org 4.3 progress report; Godot stable docs — 검증 3-0)
- 조치: Godot Web export preset에서 **Thread Support 끄고 재익스포트**.
- 효과: Vercel 헤더 설정·R2 CORP/CORS·iframe isolation **전부 불필요**.
- trade-off: 엔진 멀티스레딩 없음(성능), 과거 웹오디오 글리치 이력. **오디오 없는 제품 뷰어면 영향 적음** → 실제 export로 1회 성능 테스트. *(open question)*

## 3. 에셋 호스팅 — Cloudflare R2
- R2 **egress 무료**: *"Egressing directly from R2 … does not incur data transfer charges"* (Cloudflare 공식 pricing — 검증 3-0).
- 저장 $0.015/GB-월, 요청 Class A $4.50/M·B $0.36/M, 무료티어 10GB+1M/10M.
- 반복 다운로드 패턴에서 전송비가 비용을 지배하는 S3/Vercel Blob 대비 **결정적으로 저렴**.
- **wasm 공유 트릭** *(검증 필요, medium)*: 엔진 wasm은 전 QR 공유 1개, QR별로는 작은 `.pck`만. Godot `preloadFile()` + `--main-pack <외부 URL>` 패턴이 지원(docs는 "directory" 표현 → cross-origin pck fetch는 소규모 스파이크로 검증).

R2 버킷 구조(예):
```
r2://poim-viewers/
  engine/viewer.wasm   (공유, Brotli, immutable)
  engine/viewer.js
  packs/{qrId}.pck     (QR별, 작음)
```

## 4. 임베드 — canvas 직접 마운트 (iframe ❌)
Godot 메인 JS가 `Engine()` 클래스 제공 → canvas override로 특정 캔버스 지정:
```js
const engine = new Engine(GODOT_CONFIG);
engine.startGame({ canvas: document.querySelector('#viewer-canvas') });
```
- **왜 iframe 아닌가**: COEP `require-corp`에선 cross-origin iframe도 자기 COEP+CORP 헤더 필요. `credentialless` 우회는 **Chromium 전용(FF/Safari ✗)** → 모바일 QR에서 깨짐 (검증 3-0). single-thread면 이 문제 자체가 사라지나, 직접 마운트가 더 단순·same-origin.
- **번들 격리**: Godot JS는 뷰어 라우트 mount 시에만 동적 로드 → 40MB가 메인 번들/타 라우트에 진입 금지.

## 5. (멀티스레드 유지 시에만) COOP/COEP 헤더
single-thread면 **이 섹션 전체 무시**. 유지 시:
- **Vercel**(`next.config.js` `headers()`): 뷰어 경로에만 스코프 — 전체 사이트에 걸면 analytics/지도/결제 third-party 임베드 깨짐.
  - `Cross-Origin-Opener-Policy: same-origin` + `Cross-Origin-Embedder-Policy: require-corp`
  - source 패턴은 `/:locale/viewer/:path*` (`[locale]` 상호작용 1회 검증 — open question).
- **R2 에셋**: `require-corp` 하 cross-origin 로드 허용 위해 `Cross-Origin-Resource-Policy: cross-origin` **또는** 정상 CORS(`Access-Control-Allow-Origin`)+로더 `crossorigin`. (MDN — 검증 3-0)
  - R2가 CORP/Brotli `Content-Encoding`을 기본 제공하는지 불확실 → **Cloudflare Worker/Transform Rule**로 헤더 부여가 안전. *(open question)*

## 6. 콜드 로딩 최적화
- **Brotli 사전압축**: 40MB → **~5MB** (Godot 공식 — 검증 3-0). `.wasm.br` + `Content-Encoding: br`.
- **immutable 캐시**: `Cache-Control: public, max-age=31536000, immutable` (파일명 해시).
- **streaming instantiation**: `instantiateStreaming` 경로여야 V8 wasm 코드캐싱 동작(웜로드 가속). Godot 로더가 이미 streaming 사용.
- **lazy load**: 뷰어 라우트 진입 시에만 엔진 로드 + progress UI.

**`/ko/viewer/poim` 에서 3D 모델 실제 렌더 확인** (헤드리스 Chrome+CDP 스크린샷, Godot v4.6.2 부팅·WebGL2·씬 로드 성공).
독립 확인용: `3d-viewer-test/Poim_3dviewer/`에 `serve.py` → `http://localhost:8000/viewer.html`.

**구현 구성:**
- 에셋: `apps/brokenegg-app/godot-assets/<id>/`(gitignore). 엔진이 직접 읽는 파일은 **무압축 서빙**(아래 ⚠️).
- 서버: `src/app/godot/[...path]/route.ts` — 같은 오리진, viewer.js/viewer_function.js 는 경로/URL 치환 후 서빙.
- 임베드: `src/widgets/godot-viewer/` — canvas 직접 마운트(iframe 없음), 스크립트 동적 주입(async=false).
  - **teardown(메모리)**: 한 탭에서 모델을 SPA 전환해도 wasm 힙/GPU 가 누적되지 않도록, 언마운트 시 `engine.requestQuit()`(메인루프 종료)+`WEBGL_lose_context`(GPU 해제)+주입 스크립트 제거. 엔진 인스턴스는 핸들러가 `window.__godotEngine` 으로 노출(viewer.js 패치). 다음 로드 시 `var Godot` 재할당으로 이전 힙이 GC 대상. (QR=매 스캔 새 페이지면 전체 리로드로 자동 해제라 무관; teardown 은 "한 탭 다중 전환" 대비.)
- 라우트: `src/app/[locale]/viewer/[id]/page.tsx` → `/ko/viewer/poim`.
- 헤더: **불필요 → 제거함.** Poim export 는 single-thread(`GODOT_THREADS_ENABLED=false`)라 COOP/COEP 없이 렌더됨.
- e2e 테스트: `apps/brokenegg-app/e2e/viewer.spec.ts` (Playwright). `pnpm --filter @brokenegg/brokenegg-app test:e2e`.
  - Playwright 로 검증: `crossOriginIsolated=false`, canvas 1280×720 WebGL2, 씬 로드 성공, 실패요청 0 → **통과**.

**⚠️ 실측으로 확정된 핵심 제약 (초기 실패→해결 과정에서 발견):**
1. **필수 파일 5개**: `viewer.js·.wasm·.pck` + **`viewer_function.js`**(전역 `prograssbar`/`downloadFileLength`+`initSetting`) + **`poim_scene.zip`**(실제 모델 데이터, 런타임 HTTPRequest 다운로드). 뒤 둘을 빠뜨리면 엔진은 떠도 씬이 안 뜸.
2. **brotli 는 `Content-Length` 없이(chunked 스트리밍) 서빙해야 함** (egress 절감 유지의 핵심): Godot 로더가 `new Uint8Array(Content-Length)` 로 버퍼를 미리 잡는데, br 사전압축본을 **Content-Length(=압축크기)와 함께** 주면 디코드크기(더 큼)와 안 맞아 **오버플로 → "Failed loading file"**. ✅ 해결: **Content-Length 를 생략하고 `ReadableStream`(chunked transfer)으로** 보내면 — 브라우저가 br 을 자동 해제하므로 — **압축 전송(wasm 37.7→7.5MB, pck 3.4→1.1MB) 유지 + 로더 호환**. `route.ts` 의 br-스트리밍 분기로 구현, Playwright 검증 완료. (이게 처음엔 "brotli 금지"로 오진했던 부분 — 진짜 원인은 brotli 가 아니라 Content-Length 불일치였음.) 단, `.zip` 은 이미 압축돼 br 효과 1.4% → 무압축 그대로.
3. **동적 주입 스크립트는 `document.currentScript`가 null** → viewer.js 가 wasm/pck 를 페이지경로 기준으로 잘못 찾음. 핸들러가 `executable` 을 절대경로로 치환해 해결.
4. **Godot HTTPRequest 는 전체 URL(scheme+host) 요구** → `initSetting.url` 을 `http://<host>/godot/<id>/` 로 치환.
5. **이 export 는 single-threaded 빌드**(`GODOT_THREADS_ENABLED=false`, 엔진 로그 "single-threaded"). → **COOP/COEP 헤더를 제거해도 정상 렌더됨을 Playwright 로 확정**(crossOriginIsolated=false에서 통과). 그래서 `next.config.js` 의 headers() 는 제거함. §2/§5 의 "멀티스레드 전제"는 export마다 `GODOT_THREADS_ENABLED`/엔진 로그로 재확인할 것 — single-thread면 헤더 불필요.

### 단일스레드 export로 전환(원복) 시 — 딱 한 곳
멀티스레드→단일스레드 재익스포트본을 받으면:
1. `godot-assets/<id>/` 교체 → `node godot-assets/compress.mjs <id>`.
2. **`next.config.js`의 `[멀티스레드 export 전용]` 마커 `headers()` 블록 전체 삭제** (COOP/COEP 불필요) → dev 서버 재시작.
3. route handler·widget·page·brotli는 **그대로** 동작. (절차 상세: `apps/brokenegg-app/godot-assets/README.md`)
- 부수효과: 레이아웃 Google Fonts의 COEP 차단 경고도 사라짐.

## 10. 용어/개념 정리 (배경 지식)

### Ingress vs Egress (비용의 핵심)
- **Ingress(인그레스)** = 스토리지로 **들어오는** 트래픽 = 내가 wasm을 업로드. 보통 어디서나 무료.
- **Egress(이그레스)** = 스토리지에서 **나가는** 트래픽 = 사용자가 뷰어를 열어 wasm을 **다운로드**. **여기가 비용을 지배**.
- QR을 N명이 스캔 = wasm이 N번 egress. S3/Vercel Blob은 egress에 GB당 과금(볼수록 ↑), **R2는 egress $0**.

### 오브젝트 스토리지 (S3 / Vercel Blob / R2)
- "파일을 통째로 올려두고 URL로 꺼내 쓰는 창고". 셋 다 본질 동일, **요금(특히 egress)만 다름**.
- **S3** = Amazon(AWS) 오브젝트 스토리지 원조. **Vercel Blob** = Vercel 제공. **R2** = Cloudflare 제공, **S3 API 호환 + egress 무료**가 차별점.
- **Blob** = "Binary Large OBject" = 그냥 "바이너리 파일 덩어리". wasm·pck·js·이미지 다 객체(blob). 익스포트 원본 파일을 **가공 없이 그대로** 올린다.
- 흐름: ①업로드(ingress) → ②사용자 요청 시 다운로드(egress) → ③**CDN이 엣지에 캐시** → 다음 사용자는 가까운 엣지에서 받음.

### Godot 파일 구성 (엔진 ↔ 콘텐츠 분리)
| 파일 | 역할 | 비유 |
|---|---|---|
| `viewer.wasm` | **엔진**(Godot 런타임 본체) | 게임기 본체 |
| `viewer.pck` | **콘텐츠**(3D 씬·모델·스크립트·텍스처 묶음) | 게임 카트리지 |
| `viewer.js` | 엔진 부팅 로더 | 전원 버튼 |
- **`.pck` = PaCK 파일** = Godot 전용 리소스 묶음. `poim_scene.zip` 안의 `.scn`(씬)·`.gdc`(스크립트)가 빌드되면 `.pck` 하나로 묶임.
- 핵심: **엔진(wasm)은 모든 QR이 동일** → wasm은 R2에 1개만, QR별로는 작은 `.pck`만 다르게(=§1 wasm 공유 트릭의 원리).

### Brotli (압축)
- Google이 만든 웹용 **압축 알고리즘**. gzip보다 압축률 ↑, 브라우저 표준 지원.
- wasm은 압축이 잘 먹음: **40MB → ~5MB(약 86%↓)**.
- 동작: 서버가 `.wasm.br` + `Content-Encoding: br` 전송 → **브라우저가 자동 해제**(앱 코드 불필요).

### 용량 개선 레버 (요약 — 겹쳐 적용)
- **(a) 엔진 wasm 공유**: `37MB 엔진 1개 + 3MB×N`. QR 10개면 67MB(개별이면 400MB). 두 번째 QR은 엔진 캐시 히트 → 즉시 로딩. ⭐
- **(b) Brotli**: 40MB→5MB.
- **(c) single-thread 재익스포트**: 멀티스레드 코드 제거로 wasm ↓.
- **(d) 미사용 모듈 끄기**: 안 쓰는 기능(텍스트 서버 등) 비활성 → 엔진 33MB→25MB 식 추가 감소(실측 권장).
