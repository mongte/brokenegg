# 브로큰에그 디자인 시스템 — 개요

랜딩 페이지(`apps/brokenegg-app`)에서 확립한 디자인 언어를 **신규 페이지(상세/하위 페이지)에서도 동일하게** 쓰기 위한 기준 문서. Variant 레퍼런스 디자인(`BRK-EGG-2024-V1`)을 코드로 옮기며 정립했다.

> 구현 단일 출처(실제 값): `apps/brokenegg-app/src/app/global.css`
> 신규 페이지 작성 절차: 전용 스킬 `/brokenegg-page`
> 같이 보기: [[디자인 시스템 - 디자인 토큰]] · [[디자인 시스템 - 컴포넌트]] · [[디자인 시스템 - 페이지 레이아웃 패턴]]

## 한 줄 컨셉
**옅은 그리드 배경 위에 떠 있는, 모서리가 둥근 흰 "canvas" 카드 안의 에디토리얼(잡지/갤러리) 레이아웃.** 색을 거의 쓰지 않고, 굵기·크기·서체(산세리프↔세리프) 대비와 1px 보더로 위계를 만든다.

## 7가지 원칙 (신규 페이지에도 그대로 적용)
1. **프레임-인-캔버스**: 페이지 전체는 옅은 회색(`--frame-bg`) 위, 중앙 정렬된 흰색 `.canvas`(max 1440px, radius 32px) 안에 들어간다. 뒤에는 고정된 `.grid-bg`(100px 격자).
2. **모노크롬 우선**: 검정/흰색/회색만 사용. 별도 액센트 컬러 없음. 강조는 **색이 아니라 굵기·크기·여백·세리프**로.
3. **두 서체의 역할 분담**: 본문·UI는 **Pretendard(산세리프)**. 크고 표현적인 순간(섹션 태그, 인용구)만 **Noto Serif KR(세리프)**.
4. **마이크로 타이포 라벨**: 작은 메타/배지는 10~11px, 700, 대문자, letter-spacing. (예: `LIVE INTERACTIVE`, `BE-01`)
5. **에디토리얼 그리드**: 섹션을 위에서 아래로 쌓고, 각 섹션 경계를 `border-top: 1px solid var(--border)`로 나눈다. 표(table)처럼 정돈된 느낌.
6. **한 번의 대비 면**: 페이지당 보통 한 번, 풀블리드 **검정 배경 미디어 그리드**(갤러리)로 시각적 리듬을 준다. 이미지는 기본 그레이스케일 → 호버 시 컬러.
7. **알약 형태**: 배지·버튼은 `--radius-sm`(100px) 알약. 캔버스·큰 카드는 `--radius-lg`(32px).

## 분위기 키워드
미니멀 · 에디토리얼 · 갤러리 · 정밀(테크니컬) · 절제 · 여백. 피해야 할 것: 그라데이션 남발, 컬러 액센트, 둥근 그림자 카드 UI, 이모지 장식, "혁신적/미래의" 류 클리셰 카피([[랜딩 페이지 카피]] 톤 규칙 참고).

## 코드 ↔ 디자인 매핑 (FSD)
- 디자인 토큰·전역 클래스 → `src/app/global.css`
- 재사용 프리미티브(Badge/Button/Logo) → `src/shared/ui/*`
- 페이지 구성 블록(Header/Hero/Footer 등) → `src/widgets/*`
- 화면(라우트 1:1) → `src/views/*`, Next 라우트는 `src/app/*`
- 자세한 배치 규칙 → [[디자인 시스템 - 페이지 레이아웃 패턴]]
